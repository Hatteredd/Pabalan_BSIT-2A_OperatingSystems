Adding DHCP Role

Activity Overview

In this activity you will add and configure the Dynamic Host Configuration Protocol (DHCP) role on a Windows server. DHCP is responsible for automatically assigning IP addresses, subnet masks, gateways, and DNS servers to clients, which is critical for networks with many devices. Without DHCP, every client would need to be configured manually, which is time-consuming and error-prone.

You will walk through installing the DHCP server role, creating and activating scopes, defining exclusions and reservations, and understanding lease duration. The goal is to see how a properly configured DHCP server can serve dozens of machines in a lab, ensuring that each client receives a correct and non-conflicting IP configuration every time it joins the network.
